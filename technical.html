Architecture

Extendo is not an orchestrator. It's the human interface layer. Three pieces, clean separation:

The brains

Your orchestrator — Beads, Antfarm, custom scripts. Decides when to ask a human.

The muscle

Your agents — Claude Code, OpenClaw, whatever. Decides what to ask.

The nerve endings

Extendo — iOS app, Watch, CLI. Decides how it feels to answer.

The CLI

Any agent, any script, any pipeline can reach a human through the Extendo CLI. It blocks until the human responds and returns structured data — not chat text.

# Send a message
extendo send "Build complete. 3 warnings."

# Binary decision — returns "yes" or "no"
extendo decide --type yes_no "Deploy to prod?"

# Multiple choice — returns the selected option
extendo decide --type multiple_choice "Which fix?" --options "Retry,Skip,Abort"

# Rank options — returns ordered list
extendo decide --type rank "Priority?" --options "Auth,Search,Billing"

# Document review — returns annotations
extendo artifact --type document_review --file spec.md

The CLI is also available as a Claude Code skill, so Claude can call it natively inside agent workflows.

Backends

Extendo uses a clean adapter interface. Each backend translates between an external service and the Extendo protocol. Writing a new backend is just implementing a TypeScript interface.
